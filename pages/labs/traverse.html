Enumeração de Diretórios, Exploração de APIs e Análise de Código: Lições do Lab "Traverse"

Publicado em 2025

Introdução

O lab "Traverse" do TryHackMe simula um ambiente web vulnerável, desafiando o participante a identificar diretórios ocultos, explorar endpoints de API inseguros e analisar código ofuscado. O objetivo é desenvolver habilidades práticas em pentest web, correlacionando pistas e explorando diferentes vetores de ataque.

O Desafio

O desafio consiste em analisar um site comprometido, identificar arquivos e diretórios relevantes, explorar APIs mal protegidas e manipular parâmetros para obter informações sensíveis e escalar privilégios.

Raciocínio e Abordagem

A abordagem deste lab exigiu técnicas de análise manual, automação e raciocínio lógico. O ponto de partida foi a análise do código-fonte da página inicial, onde scripts JavaScript ofuscados e comentários revelaram pistas sobre a estrutura do sistema. A identificação de arquivos como custom.min.js e diretórios comentados foi fundamental para direcionar a enumeração.

A enumeração de diretórios foi realizada tanto manualmente, acessando URLs sugeridas por comentários, quanto com ferramentas automatizadas. O acesso ao diretório /logs/ revelou informações sensíveis.

A exploração de endpoints de API exigiu criatividade e persistência. Manipulando parâmetros como customer_id, foi possível acessar dados de diferentes usuários, incluindo administradores. Esse processo envolveu tentativa e erro, análise das respostas da API e identificação de padrões que pudessem indicar privilégios elevados ou informações sensíveis.

A análise e deobfuscação de código JavaScript foram etapas críticas para extrair informações ocultas. Ferramentas online e conhecimento prévio sobre técnicas de ofuscação foram utilizados para decodificar scripts e identificar comandos ou credenciais embutidas.

Ferramentas, Técnicas e Conceitos Aplicados

  • Nmap: Enumeração de serviços.
  • Análise de código-fonte: Identificação de scripts e comentários relevantes.
  • Deobfuscação de JavaScript: Extração de informações de código ofuscado.
  • Enumeração de diretórios: Acesso a /logs/ e outros caminhos ocultos.
  • Manipulação de APIs: Teste de parâmetros como customer_id para obter dados sensíveis.
  • DevTools: Alteração de comandos e análise de respostas.

Descrição do Problema

O principal desafio foi correlacionar pistas espalhadas pelo sistema, identificar endpoints vulneráveis e manipular parâmetros para escalar privilégios. A análise de código ofuscado e a enumeração detalhada foram cruciais para o sucesso.

Aplicação no Mundo Real

A exploração de APIs mal protegidas é uma das principais fontes de vazamento de dados. A análise de código-fonte e a enumeração de diretórios são base em pentests web.

Solução Encontrada

  • Análise de código-fonte e identificação de scripts ofuscados.
  • Enumeração de diretórios e acesso a logs.
  • Descoberta de senhas e nomes de diretórios baseados em metodologias de desenvolvimento seguro.
  • Manipulação de APIs para obter dados de usuários e administradores.
  • Acesso ao painel administrativo e execução de comandos para coleta das flags.

Generalização do Conhecimento

As técnicas aplicadas neste lab são utilizadas em avaliações de segurança de aplicações web. A enumeração detalhada de diretórios e arquivos, análise de código-fonte (incluindo scripts ofuscados) e exploração de APIs são habilidades essenciais para profissionais de segurança ofensiva.

No mundo real, APIs mal protegidas são frequentemente alvo de ataques, pois podem expor dados sensíveis ou permitir escalada de privilégios. A capacidade de identificar endpoints ocultos, manipular parâmetros e analisar respostas é fundamental para detectar e explorar essas vulnerabilidades. Além disso, a análise de código ofuscado é uma habilidade valiosa para identificar backdoors, comandos ocultos ou credenciais embutidas em aplicações web.

Reflexão e Recomendações

  • Correlacionar pistas e explorar diferentes vetores de ataque.
  • Analisar cuidadosamente código-fonte e comentários.
  • Testar endpoints de API para manipulação de parâmetros.
  • Praticar deobfuscação de código e análise de logs.
