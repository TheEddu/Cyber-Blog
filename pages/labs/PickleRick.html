Enumeração, Exploração Web e Escalonamento de Privilégios: Lições do Lab Pickle Rick

Publicado em 2025

Introdução

O laboratório Pickle Rick do TryHackMe é um desafio introdutório que simula um ambiente Linux vulnerável. O objetivo é obter 3 flags escondidas em diferentes níveis de acesso, praticando técnicas essenciais de enumeração, exploração de aplicações web e escalonamento de privilégios. Este artigo detalha o raciocínio, ferramentas e aprendizados do lab, destacando como essas lições se aplicam ao mercado real.

O Desafio

O desafio consiste em acessar uma máquina Linux, identificar serviços expostos, explorar uma aplicação web vulnerável, obter acesso ao sistema e escalar privilégios para capturar todas as flags. O participante deve aplicar técnicas de reconhecimento, análise de código-fonte, exploração de comandos web e manipulação de permissões.

Raciocínio e Abordagem

A resolução do lab começou pela enumeração de portas com Nmap, identificando SSH e HTTP abertos. O foco inicial foi a aplicação web na porta 80, onde um formulário de login sugeria a existência de credenciais expostas. A análise do código-fonte revelou o nome de usuário, enquanto o arquivo robots.txt forneceu uma possível senha.

Após o login, o painel web permitia a execução de comandos arbitrários, facilitando a exploração do sistema. Com comandos básicos de Linux (ls, cd, pwd), foi possível localizar arquivos de flag e explorar diretórios de usuários.

Para obter a flag final, foi necessário escalar privilégios utilizando o comando sudo, que estava mal configurado e permitia acesso ao diretório root sem senha.

O raciocínio aplicado envolveu a constante análise de cada resposta do sistema, correlacionando informações encontradas em arquivos, comentários e funcionalidades da aplicação. A enumeração de diretórios e arquivos foi realizada de forma sistemática, explorando diferentes caminhos e testando comandos para mapear a estrutura do sistema. A criatividade foi importante para identificar formas alternativas de acessar arquivos protegidos, como o uso de sudo para contornar restrições de permissões.

O uso de ferramentas básicas do Linux e a compreensão do funcionamento do sistema de arquivos também foram essenciais para navegar pelo ambiente e localizar as flags.

Ferramentas, Técnicas e Conceitos Aplicados

  • Nmap: Varredura de portas e serviços.
  • Reconhecimento web: Análise de formulários, código-fonte e arquivos como robots.txt.
  • Enumeração de diretórios e arquivos: Uso de comandos básicos do Linux via painel web.
  • Execução de comandos remotos: Exploração de funcionalidades perigosas em aplicações web.
  • Escalonamento de privilégios: Uso indevido de permissões sudo para acessar arquivos protegidos.

Problemas e Dificuldades

O principal desafio foi identificar informações sensíveis em locais não convencionais, como comentários no código-fonte e arquivos de configuração. A exploração do painel de comandos exigiu criatividade para navegar pelo sistema e localizar as flags. A escalonamento de privilégios dependia da identificação de permissões inadequadas no sudoers.

Aplicação no Mundo Real

No mercado, a análise de arquivos como robots.txt e o reconhecimento de informações expostas em código-fonte são práticas essenciais em pentests. Funcionalidades de execução de comandos em aplicações web representam riscos críticos, frequentemente explorados por atacantes para obter acesso ao sistema.

O escalonamento de privilégios via sudo mal configurado é uma vulnerabilidade comum em ambientes Linux, podendo resultar em comprometimento total do sistema.

Solução e Generalização

A solução envolveu a integração de técnicas de enumeração, exploração web e manipulação de permissões. O raciocínio lógico e a atenção a detalhes foram fundamentais para o sucesso. Além da capacidade de correlacionar informações, testar hipóteses e adaptar a abordagem conforme o contexto.

Reflexão e Recomendações

  • Realizar enumeração detalhada de serviços e arquivos.
  • Analisar cuidadosamente código-fonte e arquivos de configuração.
  • Restringir funcionalidades perigosas em aplicações web.
  • Configurar corretamente permissões de sudoers em sistemas Linux.
  • Praticar o raciocínio lógico e a criatividade na resolução de problemas.
